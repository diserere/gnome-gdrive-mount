# gnome-gdrive-mount

Небольшая утилита для Linux/GNOME, которая монтирует Google Drive через `rclone` как обычную локальную папку и добавляет в меню приложений лаунчер для быстрого подключения и отключения.

## Возможности

- Монтирование удалённого хранилища `gdrive:` в каталог `~/GoogleDrive` командой `rclone mount --vfs-cache-mode writes`.
- Идемпотентное повторное подключение: если диск уже смонтирован, скрипт просто сообщает об этом.
- Безопасное размонтирование через `fusermount -u` с понятной диагностикой, если какие-то файлы ещё открыты.
- Удаление точки монтирования только если она пуста (`rmdir`, а не `rm -rf`) — локальные файлы не пострадают.
- Защита от перекрытия локальных данных: если каталог существует и не пуст, монтирование отказывается.
- Быстрый возврат управления: `rclone` запускается в фоне, скрипт завершается за ~1 секунду, поэтому `.desktop`-лаунчер не «висит».
- Интеграция с GNOME: лаунчер в меню приложений с действием «Отключить диск» в контекстном меню.

## Требования

- `rclone` с настроенным удалённым хранилищем с именем `gdrive:` (создаётся через `rclone config`).
- `fusermount` (пакет `fuse`) — для размонтирования.
- `mountpoint` (пакет `util-linux`) — для проверки состояния монтирования.

## Установка

1. Сделайте скрипт исполняемым:
   ```bash
   chmod +x src/gdrive-mount
   ```
2. Установите скрипт в пользовательский каталог `bin`:
   ```bash
   install -m 755 src/gdrive-mount ~/.local/bin/gdrive-mount
   ```
3. Установите `.desktop`-файл, чтобы лаунчер появился в меню приложений GNOME:
   ```bash
   install -m 644 src/rclone-gdrive.desktop ~/.local/share/applications/
   ```
4. Убедитесь, что `~/.local/bin` присутствует в `PATH` (обычно GNOME добавляет его автоматически).

## Использование

```bash
gdrive-mount          # смонтировать gdrive: в ~/GoogleDrive
gdrive-mount -u       # размонтировать
gdrive-mount -h       # справка
```

Из меню приложений GNOME: запустите «Google Drive» для подключения, правый клик → «Отключить диск» для размонтирования.

## Структура репозитория

```
src/
  gdrive-mount             # bash-скрипт — логика монтирования/размонтирования
  rclone-gdrive.desktop    # GNOME-лаунчер с действием «Отключить диск»
```

Системы сборки, пакетирования, тестов и линтеров в проекте нет — поставляются только скрипт и `.desktop`-файл.

## Поведение и коды возврата

- `0` — успех или идемпотентная повторная операция (например, диск уже смонтирован).
- `1` — реальная ошибка: не удалось размонтировать (вероятно, открыты файлы), точка монтирования не пуста или `rclone` не смог подключиться.

## Важные замечания

- Точка монтирования `~/GoogleDrive` и имя удалённого хранилища `gdrive:` зашиты в скрипте. При изменении этих значений сохраняйте значения по умолчанию, чтобы не сломать уже установленные `.desktop`-файлы, вызывающие `gdrive-mount` без аргументов.
- Скрипт намеренно не использует `set -e`: ошибки обрабатываются явными `if`-проверками с понятными сообщениями пользователю.
